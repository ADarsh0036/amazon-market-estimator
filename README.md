# 📦 Amazon Market Estimator

Paste any Amazon Best Sellers URL and instantly get an AI-powered estimate of monthly market size, competition levels, and entry opportunities.

> **Built for:** [Pixii](https://pixii.ai) — an Amazon seller tools company, as part of a technical assignment.

## Features

- **Market size estimation** — estimates monthly unit sales and revenue for the top 10 products in any Best Sellers category
- **AI market intelligence** — a 3-sentence market summary generated by Claude: who dominates, the best price band to enter, and one actionable recommendation
- **Market gap finder** — identifies price buckets (e.g. $15–$30) with high demand but low competition, scored by entry difficulty (Easy / Moderate / Hard)
- **Competition heatmap** — ranks each product LOW → VERY HIGH based on review count, with a per-product entry difficulty score out of 10
- **Opportunity scores** — composite score (0–100) per product, weighted by BSR rank, review count, and price range
- **ENTER / WATCH / AVOID recommendations** — per-product verdict with a plain-English reason
- **Revenue bar chart** — horizontal bar chart of estimated monthly revenue across the top 10
- **Product detail modal** — deep-dive popup with price, sales, revenue, competition level, review velocity, and suggested undercut price
- **CSV export** — one-click export of the full analysis table, named by category and date
- **Animated progress steps** — real-time step indicator while the scrape and AI analysis run
- **Responsive UI** — works on desktop and mobile

## How It Works

1. The user pastes an Amazon Best Sellers URL (e.g. `amazon.com/Best-Sellers-Electronics/zgbs/electronics/`)
2. The backend proxies the page through **ScraperAPI** to bypass Amazon's bot detection
3. **Cheerio** parses the HTML and extracts the top 10 products: name, price, star rating, and review count
4. The product list is sent to **Claude (`claude-haiku-4-5`)**, which uses BSR-position heuristics to estimate monthly sales and revenue per product:
   - BSR #1 in a major category ≈ 3,000–5,000 units/month
   - BSR #10 ≈ 1,500–2,500 units/month
   - Adjusted for price, rating, and review velocity
5. The backend then locally computes competition levels, opportunity scores, market gap buckets, and brand extraction
6. Two additional Claude calls generate gap insights and a 3-sentence market summary
7. All results are returned to the frontend and rendered as a tabbed dashboard

## Setup

### 1. Clone the repo

```bash
git clone https://github.com/ADdarsh2131/amazon-market-estimator.git
cd amazon-market-estimator
```

### 2. Install backend dependencies

```bash
cd backend
npm install
```

### 3. Add environment variables

Create a `.env` file in the **project root** (next to the `backend/` folder):

```
SCRAPER_API_KEY=your_scraperapi_key_here
ANTHROPIC_API_KEY=your_anthropic_api_key_here
```

### 4. Start the backend

```bash
cd backend
npm start
```

The server starts on `http://localhost:3000`.

### 5. Open the frontend

Open `frontend/index.html` directly in your browser. No build step or local server needed.

## API

### `POST /api/analyze`

Scrapes the given Amazon Best Sellers URL and returns a full market analysis.

**Request body:**
```json
{ "url": "https://www.amazon.com/Best-Sellers-Electronics/zgbs/electronics/" }
```

**Response:**
```json
{
  "success": true,
  "url": "...",
  "products": [ { "rank": 1, "name": "...", "price": 29.99, "estimated_monthly_sales": 4500, "estimated_monthly_revenue": 134955, "confidence": "high", "competition_level": "HIGH", "entry_difficulty_score": 7, "opportunity_score": 62, "recommendation": "WATCH", ... } ],
  "marketGaps": [ { "price_range": "$15–$30", "avg_revenue": 85000, "entry_difficulty": "Easy", "insight": "..." } ],
  "market_summary": "Three-sentence AI summary...",
  "totalMonthlyRevenue": 1200000,
  "scrapedAt": "2026-05-03T10:00:00.000Z"
}
```

### `GET /health`

Returns `{ "status": "ok" }`. Use to confirm the server is running.

## External APIs Used

**ScraperAPI** — handles rotating proxies and JavaScript rendering to reliably fetch Amazon pages without being blocked. Each `/api/analyze` call makes one ScraperAPI request.

**Anthropic Claude API (`claude-haiku-4-5`)** — used for three tasks per request:
1. Estimating monthly sales and revenue for each product (structured JSON output)
2. Generating one-sentence insights per market gap
3. Writing a 3-sentence market summary with actionable recommendations
